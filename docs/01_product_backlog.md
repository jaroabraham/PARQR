# Product Backlog — PARQR

Este documento contiene la lista priorizada de todas las funcionalidades requeridas para el sistema de control de acceso vehicular PARQR.

## Historias de Usuario (Alta Prioridad)
1. **PBL-01:** Como administrador, quiero un entorno de desarrollo con Docker para que el despliegue sea consistente.
2. **PBL-02:** Como residente, quiero registrarme e iniciar sesión para gestionar mis pases de invitado.
3. **PBL-03:** Como administrador, quiero aprobar o rechazar nuevos residentes para garantizar la seguridad del complejo.
4. **PBL-04:** Como residente, quiero un código QR personal y seguro para entrar al fraccionamiento.
5. **PBL-05:** Como residente, quiero generar códigos QR temporales para mis invitados.

## Historias de Usuario (Media Prioridad)
6. **PBL-06:** Como guardia, quiero un simulador de escáner que me permita validar códigos QR y ver el estado de la pluma.
7. **PBL-07:** Como administrador, quiero ver un historial detallado de todos los accesos (exitosos y fallidos).
8. **PBL-08:** Como residente, quiero poder actualizar mi perfil y contraseña.
9. **PBL-09:** Como desarrollador, quiero que el sistema use MySQL para persistencia robusta de datos.

## Historias de Usuario (Baja Prioridad / Mejora Continua)
10. **PBL-10:** Como administrador, quiero que el ESP32 se reconecte automáticamente al WiFi si falla la red.
11. **PBL-11:** Como usuario, quiero que la interfaz sea totalmente responsiva para usarla desde mi móvil.
12. **PBL-12:** Como equipo de desarrollo, queremos un pipeline de CI/CD para automatizar pruebas y despliegues.

---
*Nota: Este backlog se refina al inicio de cada sprint siguiendo el ciclo de mejora continua.*
